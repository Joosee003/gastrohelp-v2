Confianza y protección

Seguridad

Principios y medidas aplicadas para proteger el acceso y los datos gestionados por GastroHelp.

1. Acceso y autenticación

El acceso al panel requiere autenticación. Los permisos se asignan según el rol y el restaurante asociado.

2. Protección de datos

Las comunicaciones se realizan mediante HTTPS. Se aplican controles de acceso y políticas de base de datos para limitar la lectura y modificación de información.

3. Demo pública

La demostración utiliza datos ficticios y funciona en modo de solo lectura. Las acciones de escritura se bloquean tanto en la interfaz como mediante protecciones adicionales en la base de datos.

4. Proveedores tecnológicos

GastroHelp utiliza proveedores especializados de infraestructura, alojamiento y base de datos. El acceso se limita a lo necesario para prestar y mantener el servicio.

5. Gestión de incidencias

Las incidencias de seguridad se revisan y corrigen según su impacto. Las credenciales nunca deben compartirse por mensajes o canales no seguros.

6. Comunicación responsable

Para informar de una posible vulnerabilidad, escribe a gastrohelpsmart@gmail.com incluyendo una descripción clara y pasos para reproducirla. No accedas a datos de terceros ni interrumpas el servicio.

Última actualización: 23 de julio de 2026. Esta página describe las medidas generales y no constituye una certificación formal de seguridad.
